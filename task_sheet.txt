Important Notes for Developer

Stack Divergence from Signed BRD
The client-facing BRD specifies Node.js + Supabase for the backend. This task sheet uses Python / FastAPI + PostgreSQL as directed by ArgosMob internally. Ensure the client (Mr. Ajay) is informed of this change so there are no expectation gaps at delivery or during future maintenance handoff.

Never Hardcode — Ever
All API base URLs, secret keys, payment gateway keys, FCM server keys, WhatsApp API tokens, SMS provider credentials MUST live in .env
Commit .env.example with placeholder values; .env is gitignored
No rupee symbol in code strings — use 'INR' prefix

Database Rules
Never delete records — use soft delete (is_deleted BOOLEAN, deleted_at TIMESTAMP)
All monetary values stored as INTEGER in paise (not FLOAT) to avoid floating point errors
Every table must have created_at and updated_at with auto-timestamp triggers
Foreign key constraints enforced at DB level, not just application level

Security Non-Negotiables
JWT access tokens: 15 min expiry. Refresh tokens: 7 days, single-use, stored hashed in DB
OTP: stored as bcrypt hash, never plaintext; 5-min TTL enforced at DB level via expires_at column
All file upload endpoints: validate MIME type server-side (not just extension); reject executables
RBAC enforced at service layer; never rely solely on frontend to hide routes
Payment webhook: verify Razorpay signature before processing; reject unverified webhooks with 400

Performance Targets (as per BRD)
API response time < 2 seconds for standard requests
App load time < 3 seconds on 4G network
Backend must handle 10,000+ concurrent PostgreSQL connections (Supabase/pgBouncer configured)
99.5% uptime target — configure health check endpoint for hosting provider monitoring
